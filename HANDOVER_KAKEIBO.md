# Kakeibo App Handover Notes (2026-05-13)

## 現状の課題と到達点
ユーザーがシークレットモード（初期状態）でログインした際に、クラウド上の既存データとローカルのデフォルト設定が混ざる、あるいはデータが消失するという深刻なバグに対処しました。

## 実装済みの重要ロジック (v7.0)
1. **マージ判定の厳格化**:
   - `localStorage` が空、または取引が0件の場合は「初期復元モード」として動作。
   - クラウドにあるデータを100%正解とし、ローカルのデフォルトカテゴリーなどを**完全に破棄**してから同期します。
2. **クラウドログ (Audit Log)**:
   - スプレッドシート内の `logs` シートに、同期の判定理由と結果を毎時記録します。
3. **データ消失ガード**:
   - クラウドにデータがあるのにマージ結果が0件になるような異常事態では、同期を強制停止します。
4. **アトミック更新の模索**:
   - `batchClear` と `batchUpdateValues` による2段階更新。

## 引き継ぎ後の注力ポイント
- **データの整合性チェック**: ユーザーが現在抱えている「重複したカテゴリー」の解消。
- **同期の安定化**: Google Sheets API の Quota (60/min) 回避のためのバッチ処理のさらなる最適化。
- **競合解決**: 複数端末での同時編集時のコンフリクト解消（現在は Last Write Wins）。

## 参照ファイル
- `js/store/SyncManager.js`: 同期ロジックの本体。
- `js/auth.js`: Google API 呼び出し層（batchUpdate等を追加済み）。
- `TECHNICAL_REPORT.md`: 詳細な不具合分析レポート。
