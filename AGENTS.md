# Kakeibo App — AI エージェント向け指示書

このファイルは Antigravity が **コードを修正する前に必ず読むべき** 重要なルール集です。

## ⚡ 修正前の必須確認事項（3ステップ）

### Step 1: 修正範囲を限定する
`docs/ARCHITECTURE.md` を開き、**修正するファイルの責任範囲**を確認する。
修正対象ではないファイルには**絶対に手を加えない**。

### Step 2: 保護機能を確認する
`docs/PROTECTED_FEATURES.md` を開き、今回の修正が以下の機能を破壊しないか確認する：
- 【#1】ピン留め機能
- 【#2】口座・カテゴリ並び替え機能
- 【#3】残高計算ロジック
- 【#4】クラウド同期のデータ列構造
- 【#5】ID補完・データ移行ロジック

### Step 3: 最小限の変更にとどめる
修正は「依頼された箇所のみ」に集中する。
「ついでにリファクタリング」「コードを整理」などは**ユーザーの明示的な承認なしに行わない**。

## プロジェクト構成の概要

```
kakeibo_app/
├── index.html          # エントリーポイント（変更頻度: 低）
├── js/
│   ├── app.js          # ナビゲーション・初期化（変更頻度: 低）
│   ├── store.js        # データ管理の核心（変更頻度: 中）
│   ├── auth.js         # Google認証（変更頻度: 低）
│   ├── data.js         # 静的定数（変更頻度: 低）
│   └── screens/
│       ├── input.js    # 入力画面（変更頻度: 高）
│       ├── dashboard.js# ダッシュボード（変更頻度: 中）
│       ├── history.js  # 履歴画面（変更頻度: 中）
│       ├── analysis.js # 分析画面（変更頻度: 高）
│       └── settings.js # 設定画面（変更頻度: 高）
├── css/style.css       # スタイル定義
└── docs/
    ├── spec.md         # 製品仕様書（Planner が生成）
    ├── progress.md     # 開発進捗（Generator が更新）
    ├── ARCHITECTURE.md # 構造定義（参照専用）
    └── PROTECTED_FEATURES.md # 保護ロジック（参照専用）
```
