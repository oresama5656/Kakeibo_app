# 自律開発サイクル・ガイド (Autonomous Development Guide)

このプロジェクトでは、Antigravity のグローバルナレッジに保存された3つの専門役割（Planner, Generator, Evaluator）を活用して開発を進めることができます。

## 🚀 基本のワークフロー

### 1. 企画・設計 (Planner)
新しいアプリの構想や機能追加の設計を行います。
- **コマンド**: `/planner [作りたいもののアイデア]`
- **成果物**: `docs/spec.md` (製品仕様書)

### 2. 実装 (Generator)
仕様書に従って、1スプリント（1工程）ずつコードを書きます。
- **コマンド**: `/generator`
- **成果物**: 実装コード、`docs/progress.md` (進捗管理表)

### 3. テスト・評価 (Evaluator)
実装されたコードを実際にブラウザで動かしてテストします。
- **コマンド**: `/evaluator`
- **成果物**: `docs/feedback/sprint-N.md` (合否判定とフィードバック)

---

## 🔄 開発ループの回し方

1.  `/planner` で仕様書を作成する。
2.  内容を確認し、問題なければ `/generator` で Sprint 1 を実装する。
3.  実装が終わったら `/evaluator` を実行する。
    - **合格 (PASS)** 判定なら、次回の `/generator` で Sprint 2 に進む。
    - **不合格 (FAIL)** 判定なら、次回の `/generator` で修正を行う（自動でフィードバックが読み込まれます）。
4.  全スプリントが完了するまで繰り返す。

## 📂 主要なドキュメント
- `docs/spec.md`: プロジェクトの「憲法」です。
- `docs/progress.md`: 現在どのスプリントにいるか、どんな技術判断をしたかの記録です。
- `docs/feedback/`: テスト結果の履歴です。

---
> **Tip**: このガイドはグローバル設定に基づいています。新しいプロジェクトでも同様のコマンド（`/planner` 等）がそのまま使えます。
