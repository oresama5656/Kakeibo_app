# ダークモード完全対応・実装計画 (implementation_plan.md)

## 課題
プレミアム・リデザイン（履歴・分析画面）において、深縹色などの特定のカラーをハードコードしていたため、テーマを切り替えても色が固定され、視認性が低下する。

## 解決策
ハードコードされた色をCSS変数（Custom Properties）に置き換え、ダークモード時に最適な配色に自動で切り替わるようにする。

## 1. 共通カラー変数の整理 (style.css)
- `--color-premium-card`: プレミアム画面のメインカラー。
- `--color-premium-text`: プレミアム画面のテキストカラー。
- `--color-premium-accent`: プレミアム画面のアクセントカラー。
これらを `:root` と `[data-theme="dark"]` / `@media (prefers-color-scheme: dark)` で個別に定義する。

### [新配色案]
| モード | カード背景 | テキスト | グラデーション |
| :--- | :--- | :--- | :--- |
| **ライト** | #ffffff | #1a2a4d (濃紺) | #2a4073 → #1a2a4d |
| **ダーク** | #1e1e35 | #e8e8f0 (淡灰) | #3a538a → #2a4073 (少し明るく) |

## 2. 履歴・分析画面の再スタイリング
- `total-summary-card`, `bs-total-card` などの背景色を変数化。
- `history-asset-amount` などの文字色を変数化。

## 3. 検証
- 設定画面から「ダーク/ライト/自動」を切り替え、すべての画面が適切に変化することを確認。
