# 家計簿アプリ リアルタイム同期＆プライバシー強化 実装計画書 (Implementation Plan)

## 🎯 プロジェクト基本方針（Core Philosophy）
本プロジェクトの最優先事項は、**「ユーザーの財務データ（家計簿の内容）に、アプリ開発者（オーナー）が一切アクセスできない仕組みを死守する」**ことです。
この強力なプライバシー保護（Googleドライブへの直接保存）という強みを維持しつつ、ユーザー体験を損なう「データ消失リスク」「1時間ごとの再ログイン」「デバイス間の同期ラグ」を解消するための3ステップアーキテクチャを実装します。

---

## 🛠️ フェーズ1: 擬似リアルタイムと安全装置の実装（フロントエンド改修）
**担当エージェント:** `@frontend-specialist`

現在のVanilla JS（プレーンなJavaScript）の構成を維持したまま、データ消失事故を100%防ぐ安全装置（Safety Fallback）をフロントエンドに実装します。

### 1.1 タスク内容
1. **`visibilitychange` イベントの導入**:
   - ユーザーがブラウザのタブを切り替えたり、スマホのスリープから復帰したりして、アプリが再び画面に表示された瞬間（`visibilitychange` が `visible` になった時）を検知します。
   - 検知時にGoogle Sheets APIを叩き、最新のデータを取得して画面を更新します（これにより**擬似的なリアルタイム同期**が実現します）。
2. **リード・モディファイ・ライト（Read-Modify-Write）の徹底**:
   - データを保存する `save()` 処理を改修します。
   - ローカルのデータをいきなりクラウド（スプレッドシート）に上書き保存するのではなく、**必ず直前にクラウドから最新データを「読み込み（Read）」、ローカルの変更分と「合体させ（Modify / `mergeData()`）」、その上で「書き込む（Write）」**フローにします。
   - これにより、通信エラー等でローカルデータが空だった場合に、クラウドのデータを消してしまう事故を防ぎます。

### 1.2 検証方法（CEOによる確認事項）
- オフライン状態でデータを追加後、オンライン復帰時に既存データが上書き消失しないこと。
- 別タブでスプレッドシートを直接編集し、アプリのタブに戻った瞬間にデータが画面に反映されること。

## 🚀 フェーズ2: セッション永続化（マイクロバックエンド導入）
**担当エージェント:** `@backend-specialist`

GoogleのOAuth（認証）トークンが1時間で切れてしまい、ユーザーに再ログインを強いるUX上の問題を解決します。

### 2.1 タスク内容
1. **Cloudflare Workers の構築**:
   - 無料かつ高速なエッジコンピューティング環境であるCloudflare Workersを使用して「マイクロバックエンド（小規模な中継サーバー）」を構築します。
   - ※**用語解説**: エッジコンピューティングとは、ユーザーに物理的に近いサーバーで処理を行う技術で、非常に高速に動作します。
2. **リフレッシュトークン（`refresh_token`）の安全な管理**:
   - Googleから発行される、有効期限が切れない「リフレッシュトークン」を、CloudflareのKV（Key-Valueストア）等に暗号化して安全に保管します。
   - ユーザーがアプリを開いた際、フロントエンド（`auth.js`）はWorkersにアクセスし、WorkersがGoogle APIから新しい一時トークン（`access_token`）を取得してフロントエンドに渡します。
3. **ゼロ知識アーキテクチャの徹底**:
   - サーバーは「鍵の受け渡し」だけを行い、ユーザーの家計簿データそのものをログに出力したり、保存したりしない設計にします。

### 2.2 検証方法（CEOによる確認事項）
- アプリに一度ログインした後、1時間以上経過（または手動でトークンを破棄）しても、再読み込み時に自動的にログイン状態が維持されていること。

## ⚡ フェーズ3: WebSocketによる「Notion風」リアルタイム同期
**担当エージェント:** `@backend-specialist` & `@frontend-specialist`

デバイスAで入力した内容が、リロードなしで即座にデバイスBの画面に反映される「真のリアルタイム同期」を実現します。

### 3.1 タスク内容
1. **WebSocket 通信の導入**:
   - Cloudflare Workers (Durable Objectsなど) を拡張し、WebSocketサーバーを立ち上げます。
   - ※**用語解説**: WebSocketとは、ブラウザとサーバー間で「繋がりっぱなしのトンネル」を作り、データを双方向にリアルタイムで送り合える技術です。
2. **更新のブロードキャスト（一斉送信）**:
   - デバイスAがデータを追加・編集した際、変更内容（差分）をサーバーに送信します。
   - サーバーは同じユーザーが接続している別のデバイス（デバイスBやPC）に対して、即座に「ここが変更されたよ」という通知をプッシュします。
3. **非同期書き込み（キューイング）**:
   - Google Sheets APIは短時間の連続アクセス（レートリミット）に弱いため、サーバー側で書き込み要求を少しだけため込み（バッファリング）、数秒ごとにまとめてGoogle Sheetsに書き込む処理を実装します。

### 3.2 検証方法（CEOによる確認事項）
- PCとスマホの両方でアプリを開き、スマホで入力した直後にPCの画面が自動で書き換わること。
