# プロジェクト仕様書 (Specification)

## 概要
本プロジェクトは、肥大化した JavaScript アプリケーションを機能単位（コンポーネント）に分割し、AI による開発でもデグレードが発生しない堅牢な構造を構築することを目的とする。

## 1. コア機能定義
本アプリが提供する主要な機能は以下の通りである。

### 1.1 収支入力機能 (`input.js`)
- 支出、収入、振替の各取引を、日付・金額・カテゴリ・口座・メモと共に記録する。
- 数値の入力は数値キーパッドまたは直接入力で行う。
-

### 1.2 ダッシュボード (`dashboard.js`)
- 各口座の現在残高を一覧表示する。
- 今月の収支サマリーを表示する。

### 1.3 履歴管理 (`history.js`)
- 記録された取引を時系列で表示し、編集・削除が可能である。

### 1.4 分析・レポート (`analysis.js`)
- カテゴリ別の支出割合をグラフ表示する。
- 口座別の残高推移を可視化する。
- チェックボックスには一括チェックボックスもある。

### 1.5 設定・マスタ管理 (`settings.js`)
- 口座、カテゴリ、ショートカットの追加・編集・削除を行う。
- ドラッグ&ドロップによる並び替え（表示順序の変更）が可能である。
- ピン留め設定（重要項目の上位固定）が可能である。

## 2. UI/UX 要件
- **シンプルさ**: 直感的なモバイルファーストのデザイン。
- **レスポンシブ**: PC、タブレット、スマホの各解像度で適切に表示される。
- **フィードバック**: アクション完了時にトースト通知などでユーザーに報知する。

## 3. 技術的制約・ルール
- **Vanilla JS**: 外部フレームワーク（React, Vue等）を一切使用せず、標準の JavaScript (ES Modules) を使用する。
- **CSS**: Vanilla CSS（標準CSS）を使用する。
- **データ永続化**: 通常は `localStorage` に保存し、Google Sheets API を通じてクラウドと同期する。
- **デグレ防止**: あらゆる修正は `docs/PROTECTED_FEATURES.md` のコアロジックを遵守しなければならない。
