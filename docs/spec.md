# 家計簿アプリ プロジェクト仕様書 (spec.md)

## 概要
プレミアム・リデザインおよびダークモード完全対応。
肥大化した JavaScript アプリケーションを機能単位（コンポーネント）に分割し、AI による開発でもデグレードが発生しない堅牢な構造を構築する。

## 1. コア機能定義
### 1.1 収支入力機能 (`input.js`)
- 支出、収入、振替の各取引を、日付・金額・カテゴリ・口座・メモと共に記録する。
- 数値の入力は数値キーパッドまたは直接入力で行う。
- 入力中および完了後に残高が表示されるリアルタイム・フィードバック機能を備える。

### 1.2 ダッシュボード (`dashboard.js`)
- 各口座の現在残高を一覧表示する。
- 今月の収支サマリーを表示する。

### 1.3 履歴管理 (`history.js`)
- 記録された取引を時系列で表示し、編集・削除が可能。
- ダークモードへの完全適応。

### 1.4 分析・レポート (`analysis.js`)
- カテゴリ別の支出割合をグラフ表示。
- 口座別の残高推移を可視化。
- ダークモード対応のチャート表示。

### 1.5 設定・マスタ管理 (`settings.js`)
- 口座、カテゴリ、ショートカットの追加・編集・削除。
- ドラッグ&ドロップによる並び替え、ピン留め設定。

## 2. デザイン・UI/UX 要件
- **ダークモード完全対応**:
    - カラーの抽象化: CSS変数による一括管理。
    - プレミアム・質感: 深いネイビーを基調とした「プレミアム・ダーク」。
- **レスポンシブ**: PC、モバイルの各解像度で適切に表示。
- **フィードバック**: アクション完了時のトースト通知。

## 3. 技術적制約・ルール
- **Vanilla JS & CSS**: 標準技術のみを使用。
- **データ永続化**: `localStorage` + Google Sheets API 同期。
- **デグレ防止**: `docs/PROTECTED_FEATURES.md` の遵守。
