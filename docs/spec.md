# 製品仕様書 (spec.md) - カテゴリ表示ロジックの改善

## 1. 概要
入力画面におけるカテゴリおよび口座の表示ロジックを改善し、ピン留めされたアイテムがない場合でもユーザーが直感的に操作できるようにする。

## 2. 現状の課題
- ピン留めカテゴリが0件の場合、非展開状態でも最初の12件が強制的に表示される。
- ピン留めカテゴリが0件の場合、「▼ 全表示」ボタンが表示されず、リストを閉じることができない。
- ユーザーが意図しないカテゴリが常に表示されるため、画面が煩雑。

## 3. 望ましい挙動（仕様）
- **ピン留めあり**: 従来通り、ピン留めされたアイテムを表示し、余剰があれば「▼ 全表示」ボタンを出す。
- **ピン留めなし（未選択時）**: 
  - 非展開状態ではアイコンを一つも表示しない、または「選択してください」といったメッセージのみにする。
  - 常に「▼ 全表示」ボタンを表示する。
- **ピン留めなし（選択中）**:
  - 非展開状態では、現在選択されているアイテム（1件）のみを表示する。
  - 常に「全表示/閉じる」ボタンを表示する。

## 4. 影響範囲
- `js/screens/input.js` の `renderIconGrid` 関数。
- 口座選択（出金元・入金先）およびカテゴリ選択のUI。

## 5. UIデザイン
- 非展開時：[選択済みアイコン] [▼ 全表示]
- 展開時：[全アイコンリスト] [▲ 閉じる]
