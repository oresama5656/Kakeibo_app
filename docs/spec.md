# 製品仕様書 (spec.md) - 履歴画面ヘッダー改善

## 1. 概要
スマホ表示において、履歴画面のフィルターヘッダー（期間、口座、カテゴリ）が見切れてしまう問題を解消し、全ての操作項目が適切に表示されるようにする。

## 2. 現状の課題
- `.history-filters` が横並び固定（`flex-wrap` なし）のため、画面幅が狭いスマホで要素が外に食み出す。
- `input[type="date"]` や `select` の最小幅が確保されておらず、内容が判別しにくい場合がある。

## 3. 望ましい挙動（仕様）
- **レスポンシブ対応**:
  - スマホなどの狭い画面では、フィルター要素を折り返し表示（2行など）にする。
  - あるいは、各要素の幅を調整して1行に収めるか、スクロール可能にする（ユーザー要望に基づき「全部入るように」とのことなので、折り返しを優先）。
- **可読性の向上**:
  - 各入力項目の間隔を適切に保ちつつ、ボタンやセレクトボックスが押しやすいサイズを維持する。

## 4. 影響範囲
- `css/style.css` の `.history-filters` およびその子要素。
- `js/screens/history.js` のHTML構造（必要に応じてクラス追加など）。

## 5. UIデザイン案
### スマホ時
- [開始日] 〜 [終了日] （1行目）
- [全口座] [全カテゴリ] （2行目）
といった形での折り返し、または `flex-wrap: wrap` による動的なレイアウト調整。
