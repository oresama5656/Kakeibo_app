# プレミアム・デザインガイドライン (DESIGN_SYSTEM.md)

本ドキュメントは、Kakeiboアプリの「プレミアム・リデザイン」で確立されたデザイン言語を定義したものです。今後の機能追加やUI変更の際は、以下の原則を遵守してください。

---

## 1. デザイン原則：引き算の美学 (Minimalism)
- **情報の優先順位**: 最も重要な数値（合計金額など）を最大化し、それ以外の要素は余白や透過度を使って視覚的に後退させる。
- **ノイズの排除**: 境界線（Border）を減らし、影（Shadow）や背景の色の差で要素の区切りを表現する。
- **Apple風の質感**: 角丸は大きめ（16px～24px）に設定し、滑らかな質感を演出する。

## 2. カラーシステム (Premium Colors)

プレミアム画面では、CSS変数 `--premium-*` を使用し、直接の色コード指定を避けてください。

| 変数名 | ライトモード (Light) | ダークモード (Dark) | 用途 |
| :--- | :--- | :--- | :--- |
| `--premium-deep` | `#1a2a4d` (深縹) | `#0a0e1a` | メイン背景（濃）、重要テキスト |
| `--premium-mid` | `#2a4073` | `#1a2a4d` | サマリーカードのグラデーション上部 |
| `--premium-light` | `#3a538a` | `#2a4073` | サブボタン、ホバー、強調 |
| `--premium-bg` | `#ffffff` | `#1e1e35` | 画面背景、カード背景 |
| `--premium-text-on-dark`| `#ffffff` | `#e8e8f0` | 濃色背景上のテキスト |

## 3. コンポーネント設計

### 3.1 サマリー・マスター・カード
- **背景**: `linear-gradient(135deg, var(--premium-mid) 0%, var(--premium-deep) 100%)`
- **影**: `0 10px 25px var(--premium-shadow)`
- **用途**: 画面最上部での資産・合計値の表示。

### 3.2 セグメントコントロール (iOS Style)
- **背景**: `rgba(0,0,0,0.06)`（ライト）/ `rgba(255,255,255,0.06)`（ダーク）
- **アクティブ要素**: 背景を `var(--bg-card)`、文字色を `var(--premium-deep)` に。
- **用途**: 期間切り替え、表示形式のトグル。

### 3.3 プレミアム・カード (v3)
- **半径**: `var(--radius-xl)` (24px)
- **背景**: `var(--bg-card)`
- **用途**: カテゴリリスト、グラフコンテナ。

## 4. ダークモード実装ルール
- 色コードの直接指定（`#ffffff`等）は絶対に行わない。
- 常に `var(--bg-primary)` や `var(--text-primary)` などのセマンティックな変数を使用する。
- ダークモード時の背景は漆黒（#000）ではなく、深いネイビーやグレー（#1a1a2e等）を推奨。

## 5. コーディング規約
- **HTML構造**: 画面構成は `UIHelper.js` 等のコンポーネントクラスに分離し、HTMLのベタ書きを避ける。
- **セキュリティー**: すべての動的文字列（ユーザー名、口座名、カテゴリ名）は `store.escapeHTML()` を通して表示する。
