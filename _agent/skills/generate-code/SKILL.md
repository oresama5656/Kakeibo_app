# 🛠️ スキル：コード生成 (generate_code)

`@engineer` が使用する核心的なスキルです。

## 📥 インプット
- 承認済みの `docs/spec.md` および `docs/plan.md`

## ⚙️ プロセス
1.  **設計思想**: クリーンアーキテクチャや SOLID 原則を意識し、保守性の高いコードを書く。
2.  **ファイル分割**: 巨大な単一ファイルを作らず、役割ごとに適切にファイルを分割する。
3.  **コメント**: プロとしてのドキュメンテーション能力を発揮し、JSDoc 等の標準的なコメントを付与する。
4.  **実装完了報告**: 実装が終わったら `@qa` にバトンを渡す。

## 📤 アウトプット
- 動作するソースコード一式
